About The Luck Lab

The Luck Lab is an interactive 3D application designed to explore the concepts of probability and chance through engaging dice rolls and coin flips. Users can interact with various themed 3D models, change environments, and observe random outcomes in a virtual laboratory setting.

This project utilizes HTML5, CSS3, JavaScript, and the Three.js library for 3D rendering and interaction. It also uses the GSAP library for animations.

3D Models Creation Process (Blender):

All 3D models used in this application were created using Blender, focusing on applying modeling, texturing, and exporting techniques relevant to the course.

  • D6 Model (die6.glb / die6.blend):
    • Started with a default Cube mesh. Used the Bevel modifier (applied) for rounded edges.
    • UV unwrapped using Smart UV Project or manual seams.
    • Created a material using the Principled BSDF shader.
    • **Updated Appearance:** Modified material properties to achieve a **clear red** look. This involved:
      • Setting the Base Color to red.
      • Increasing the `Transmission` value significantly (towards 1.0) to allow light to pass through.
      • Adjusting `Roughness` (lower for clearer, higher for frosted) and potentially `IOR` (Index of Refraction, e.g., ~1.5 for glass/plastic).
      • Ensuring the pips/numbers were applied correctly (perhaps as separate objects or using a texture mask with the main material).
    • Exported as GLB (Apply Modifiers, UVs, Normals, Materials checked). Source `.blend` file included.
  • D12 Model (die12.glb / die12.blend):
    • Created a Dodecahedron mesh (Add Mesh > Math Function > Regular Solids or manual construction).
    • Applied a slight Bevel modifier to edges.
    • UV unwrapped carefully, marking seams to flatten pentagonal faces.
    • Created a material with a distinct base color.
    • A texture map (1-12) was created and applied to faces via the UV map.
    • Adjusted material properties (Principled BSDF).
    • Exported as GLB with embedded textures. Source `.blend` file included.
  • Coin Model (coin.glb / coin.blend):
    • Started with a Cylinder mesh, scaled flat on the Z-axis.
    • Added edge loops and Bevel modifier/manual beveling for a rounded rim.
    • Optionally used Subdivision Surface modifier (low level) for smoothness.
    • UV unwrapped (top, bottom, rim separately).
    • Created material(s) for Heads and Tails using textures/masks.
    • Designed/sourced Heads/Tails images, applied as textures to Base Color.
    • Increased Metallic property and adjusted Roughness for a coin look. Bump map optional.
    • Exported as GLB with embedded textures. Source `.blend` file included.

Testing Methodology

Browsers Tested:

  • Google Chrome (latest stable version)
  • Mozilla Firefox (latest stable version)
  • Safari (latest stable version)
  • Microsoft Edge (latest stable version)

These browsers were selected to ensure that the application functions as expected across the most commonly used platforms.

Operating Systems Tested:

  • Windows 10
  • macOS (Catalina & Big Sur)
  • Linux (Ubuntu 20.04)
  • iOS (Safari)
  • Android (Chrome)

Screen Sizes and Devices Tested:

  • Desktop (resolutions: 1920x1080, 2560x1440, 3840x2160)
  • Tablet (iPad, 768x1024, 1024x768)
  • Mobile (iPhone 12, Galaxy S20)

These screen sizes were chosen to validate the responsiveness of the design, ensuring that it is properly optimized for both small and large displays.

Testing Types Conducted

Functional Testing:

Objective: To ensure all UI elements, such as buttons, dropdowns, and input sliders, function as expected.

Tested Functionality:

  • The "Explode & Roll" button triggers the dice roll or coin flip with the correct animation.
  • The "Wireframe Toggle" button toggles between displaying the 3D model in wireframe mode and normal mode.
  • The "Glow & Spin" button activates/deactivates the glow and spin animation for 3D models.

Results: All buttons triggered the expected actions without errors.

Responsiveness Testing:

Objective: To ensure the UI is responsive and adjusts well to different screen sizes.

Tested Devices: Desktop (large, medium), Tablet (iPad), Mobile (iPhone 12).

Results: The layout adjusts fluidly across all devices. Interactive elements like buttons and sliders are accessible on both mobile and desktop views.

Cross-Browser Testing:

Objective: To ensure compatibility across popular browsers.

Tested Browsers: Chrome, Firefox, Safari, Edge.

Results: The app displayed and functioned as expected across all tested browsers. Minor styling differences were found in the handling of flexbox layouts in Safari, which was resolved with specific CSS adjustments.

Accessibility Testing:

Objective: To ensure that the application is accessible to users with disabilities.

Test Method: Used Chrome's accessibility audit tools and manually tested keyboard navigation and screen reader compatibility.

Results: The application passed basic accessibility standards. Areas for improvement include adding ARIA labels to interactive elements and adjusting color contrast for better visibility.

UI Testing:

Objective: To verify that the interface is aesthetically pleasing and elements are correctly aligned.

Tested Elements: Color schemes, fonts, button placements, alignment of models.

Results: Visual elements were rendered correctly across all screen sizes. The color scheme and font choices were consistent with the appâ€™s design goals.

Usability Testing:

Objective: To evaluate how easy it is for users to interact with the app.

Test Method: Observed user interaction with the 3D models and UI elements.

Results: Users were able to navigate through themes and models without confusion. The interactive features were intuitive, but feedback suggested that tooltips for buttons would improve understanding.

Cross-Platform Testing:

Objective: To ensure the app functions correctly across different operating systems.

Tested OS: Windows 10, macOS, iOS, Android.

Results: The app performed well across platforms. Touch gestures on mobile devices worked as expected.

Regression Testing:

Objective: To ensure that recent changes did not introduce any new bugs.

Test Method: Reviewed all previously tested functionality after adding new features.

Results: No new issues were found. The application remained stable after new additions.

Known Limitations & Issues

  • Browser-Specific Styling: There were minor visual differences in Safari, particularly in the handling of flex layouts, which have been addressed in the latest version.
  • Mobile Performance: While the app is functional on mobile devices, the performance of 3D models could be optimized further, especially on lower-end Android devices.
  • Accessibility Improvements Needed: The app lacks full keyboard navigation and ARIA labeling for buttons. These will be addressed in future updates.

Conclusion

Through a combination of functional, responsiveness, and cross-browser testing, we conducted comprehensive tests to ensure the Luck Lab application functions smoothly across multiple environments. Testing also identified areas for improvement, particularly in accessibility and performance on mobile devices. Future iterations of the app will focus on enhancing these areas while maintaining the user-friendly experience.
